# UC-02: Add a New Document to the Demo Module

**Use Case**: Add a new epistemic document to the demo module with proper validation.

## Prerequisites

- Completed UC-01 (validation works)
- Understanding of epistemic document schema
- Text editor or JSON editor

## Steps

### 1. Identify Document Content

Gather information for the new document:
- Title and description
- Domain (cartography, geodesy, remote_sensing, navigation, other)
- Document kind (map, globe, atlas, timeline, projection, survey_campaign, dataset, paper, other)
- Date range (start year, optional end year)
- Provenance (source type and reference)

### 2. Generate Document ID

**Pattern**: `{TYPE}-{SCOPE}-{IDENTIFIER}-{YEAR}.json`

**Example**: `MAP-WORLD-MERCATOR-1569.json`

- `TYPE`: MAP (for map)
- `SCOPE`: WORLD (world map)
- `IDENTIFIER`: MERCATOR (cartographer's name)
- `YEAR`: 1569 (year of creation)

### 3. Create Document File

```bash
cd modules/demo_module/documents
touch MAP-WORLD-MERCATOR-1569.json
```

### 4. Fill Required Fields

```json
{
  "type": "epistemic_document",
  "id": "MAP-WORLD-MERCATOR-1569",
  "title": "Nova et Aucta Orbis Terrae Descriptio ad Usum Navigantium",
  "domain": "cartography",
  "document_kind": "map",
  "date_range": {
    "start_year": 1569,
    "approximate": false
  },
  "provenance": {
    "source_type": "archive",
    "source_reference": "Specific archive or collection"
  }
}
```

### 5. Add Optional Fields

```json
{
  "creators": [
    {
      "name": "Gerardus Mercator",
      "role": "cartographer"
    }
  ],
  "region_scope": ["World", "Europe", "Americas"],
  "technical": {
    "projection": "Mercator projection",
    "medium": "engraved print",
    "scale": "approximately 1:20,000,000"
  },
  "claims_and_limits": {
    "commonly_stated_claims": [
      "First map to use Mercator projection",
      "Designed for maritime navigation"
    ],
    "known_limits": [
      "Extreme distortion at high latitudes",
      "Not suitable for showing relative areas"
    ]
  },
  "tags": ["projection", "navigation", "renaissance"],
  "language": "Latin",
  "created_at": "2026-03-03"
}
```

### 6. Validate the Document

```bash
cd /path/to/MWDR-Showcase
python3 src/validate/validate_json.py --path modules/demo_module/
```

**Expected Output**:
```
Validation Summary
==================
total_files_scanned: 6
parsed_ok: 6
syntax_errors: 0
schema_valid: 6
schema_invalid: 0
Result: PASS
```

### 7. Troubleshooting

#### Syntax Error

Check JSON syntax:
```bash
python3 -m json.tool modules/demo_module/documents/MAP-WORLD-MERCATOR-1569.json
```

#### Schema Validation Error

Common issues:
- Missing required field
- Wrong data type (string vs number)
- Invalid enum value
- Pattern mismatch (ID format)

Check error message in validation report for specific issue.

### 8. Commit Changes

```bash
git add modules/demo_module/documents/MAP-WORLD-MERCATOR-1569.json
git commit -m "Add Mercator world map (1569) to demo module"
```

## Checklist

Before committing, verify:

- [ ] File naming follows pattern: `{TYPE}-{SCOPE}-{IDENTIFIER}-{YEAR}.json`
- [ ] ID matches filename (without .json extension)
- [ ] All required fields present: `type`, `id`, `title`, `domain`, `document_kind`, `date_range`, `provenance`
- [ ] `date_range.start_year` is an integer
- [ ] `provenance.source_type` is valid enum value
- [ ] `provenance.source_reference` is provided
- [ ] Document validates against schema
- [ ] No syntax errors

## Tips

1. **Use existing documents as templates**: Copy and modify an existing document
2. **Validate frequently**: Run validation after each change
3. **Check enum values**: Ensure domain, document_kind, and source_type use valid enum values
4. **Date format**: Use `YYYY-MM-DD` format for `created_at`
5. **ID pattern**: Must match `^[A-Z]+-[A-Z]+-[A-Z]+-\d{4}$`
